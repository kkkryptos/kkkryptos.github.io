一、SysTick_TIM 系统滴答定时器

1、寄存器（参考手册）

1. SysTick control and status register (STK_CTRL)

Bits 31:17 Reserved, must be kept cleared.
Bit 16 COUNTFLAG: //计数标志位
**Returns 1 if timer counted to 0 since last time this was read. **//被读到就自动变0；等systick timer的专用计数器为0时标志位置1
Bits 15:3 Reserved, must be kept cleared.
Bit 2 CLKSOURCE: Clock source selection //选择时钟源（0/1）
Selects the clock source.
0: AHB/8
1: Processor clock (AHB)
Bit 1 TICKINT: SysTick exception request enable //异常请求使能
0: Counting down to zero does not assert the SysTick exception request
1: Counting down to zero to asserts the SysTick exception request.
Note: Software can use COUNTFLAG to determine if SysTick has ever counted to zero.
Bit 0 ENABLE: Counter enable //开关
Enables the counter. When ENABLE is set to 1, the counter loads the RELOAD value from the LOAD register and then counts down. On reaching 0, it sets the COUNTFLAG to 1 and optionally asserts the SysTick depending on the value of TICKINT. It then loads the RELOAD value again, and begins counting.
0: Counter disabled
1: Counter enabled

2. SysTick reload value register (STK_LOAD)

Bits 31:24 Reserved, must be kept cleared.
Bits 23:0 RELOAD: RELOAD value //用来设置计时器倒数开始的值（弹药箱）
The LOAD register specifies the start value to load into the STK_VAL register when the counter is enabled and when it reaches 0.
Calculating the RELOAD value
The RELOAD value can be any value in the range 0x00000001-0x00FFFFFF. A start value of 0 is possible, but has no effect because the SysTick exception request and COUNTFLAG are activated when counting from 1 to 0.
The RELOAD value is calculated according to its use:
l To generate a multi-shot timer with a period of N processor clock cycles, use a RELOAD value of N-1. For example, if the SysTick interrupt is required every 100 clock pulses, set RELOAD to 99.
l To deliver a single SysTick interrupt after a delay of N processor clock cycles, use a RELOAD of value N. For example, if a SysTick interrupt is required after 100 clock pulses, set RELOAD to 99.

3. SysTick current value register (STK_VAL)

Bits 31:24 Reserved, must be kept cleared.
Bits 23:0 CURRENT: Current counter value //当前计数器的值
The VAL register contains the current value of the SysTick counter.
Reads return the current value of the SysTick counter.
A write of any value clears the field to 0, and also clears the COUNTFLAG bit in the STK_CTRL register to 0.

​ //写入任何值都会将寄存器清零，同时将 STK_CTRL 里的计数标志位置0

4. SysTick calibration value register (STK_CALIB)

//校准值寄存器，当使用的时钟不是AHB，而是HCLK/8时有效，只读

​ Bit 31 NOREF: NOREF flag. Reads as zero. Indicates that a separate reference clock is provided.
​ The frequency of this clock is HCLK/8.
​ Bit 30 SKEW: SKEW flag: Indicates whether the TENMS value is exact. Reads as one. Calibration value for the 1 ms inexact timing is not known because TENMS is not known. This can affect the suitability of SysTick as a software real time clock.
Bits 29:24 Reserved, must be kept cleared.
Bits 23:0 TENMS[23:0]: Calibration value. Indicates the calibration value when the SysTick counter runs on HCLK max/8 as external clock. The value is product dependent, please refer to the Product Reference Manual, SysTick Calibration Value section. When HCLK is programmed at the maximum frequency, the SysTick period is 1ms.
​ If calibration information is not known, calculate the calibration value required from the frequency of the processor clock or external clock.

2、代码（更改时钟中断频率，观察LED1闪烁周期变化）

main() → HAL_InitTick() → HAL_SYSTICK_Config() → SysTick_Config()

可以看出传入STK_LOAD寄存器的值为HAL_SYSTICK_Config()的传参，

其值为ticks=TickNumb=SystemCoreClock / (1000U / uwTickFreq)

​ SystemCoreClock :未配置时默认为16 000 000(HSI内部高速时钟），我们在CubeMX将其配置为170 000 000（HCLK 170MHz）

​ 1000U：时钟产生中断的频率，默认为1kHz，使其周期为1ms

​ uwTickFreq：1UL

因此，ticks=170 000 000/我们想要的时钟产生中断的频率

如要使频率为10kHz：在main()中SystemClock_Config();之后调用HAL_SYSTICK_Config(17 000);

*注意*：STK_LOAD寄存器的长度为24bit=2^24-1=16777215

/*****main.c*****/

uint32_t tempCnt;

HAL_SYSTICK_Config(170000000/1000);//将1000改成10000，比较灯闪烁频率

HAL_GPIO_WritePin(LED_EN_GPIO_Port,LED_EN_Pin,GPIO_PIN_SET);//GPIO_PIN_SET高电平，使能

while (1)
{
	if(HAL_GetTick()-tempCnt>1000)
	{
		HAL_GPIO_TogglePin(LED1_GPIO_Port,LED1_Pin);
		tempCnt=HAL_GetTick();
	}
}

二、General-purpose timers (TIM2/TIM3/TIM4/TIM5)通用定时器

1、时基功能

1. 确定总线（数据手册p16）

TIMER3位于总线APB1上

2. 寄存器（参考手册p1230）

CNT counter:可增可减/先增后减（对比：SyyTick TIM只能递减计数），与Auto-reload register的阴影寄存器作比较，当Auto-reload register的值发生更改时产生更新事件，不影响当前周期，在下一次周期自动更新后生效。

阴影寄存器的事件是一定会发生的，中断通过使能后可以发生

3. CubeMX工程配置

PSC：17分频，则选择16（0-65535）

Counter Period：170 000/17 =10000

4. 代码（用TIM2时基控制LED2闪烁）

	cnt++;

/*****main.c*****/

#include "lcd.h"
#include "stdio.h"

void HAL_TIM_IC_CaptureCallback(TIM_HandleTypeDef *htim)
{
  /* Prevent unused argument(s) compilation warning */
  
	static uint32_t old_cap_cal=0;
	static uint32_t new_cap_val=0;
	
	new_cap_val=HAL_TIM_ReadCapturedValue(&htim2,TIM_CHANNEL_1); 
	
	if(new_cap_val<old_cap_cal)
	{
		pwm_period=100000-old_cap_cal+new_cap_val;
	}
	else
	{
		pwm_period=new_cap_val-old_cap_cal;
	}
	old_cap_cal=new_cap_val;

int main(void)
{

	uint32_t tempCnt;
	char disp_buff[30];
	float fre=0;

  /* USER CODE BEGIN Init */
	LCD_Init();
  /* USER CODE END Init */

  /* USER CODE BEGIN SysInit */
	HAL_SYSTICK_Config(170000000/1000);
  /* USER CODE END SysInit */

  /* USER CODE BEGIN 2 */
	HAL_TIM_IC_Start_IT(&htim2,TIM_CHANNEL_1);//打开TIM2通道1的带中断的输入捕获
	//HAL_GPIO_WritePin(LED_EN_GPIO_Port,LED_EN_Pin,GPIO_PIN_SET);//GPIO_PIN_SET高电平，使能
  /* USER CODE END 2 */

  /* USER CODE BEGIN WHILE */
  while (1)
  {

		if(HAL_GetTick()-tempCnt>100)
		{
			fre=10000000.0 / pwm_period;
		
			sprintf(disp_buff,"TIM2_CAP_Period:%.2fHz",fre);
			
			LCD_DisplayStringLine(Line6,(uint8_t*)disp_buff);
			
            tempCnt=HAL_GetTick();
		}
    /* USER CODE END WHILE */
    /* USER CODE BEGIN 3 */
  }
  /* USER CODE END 3 */
}
